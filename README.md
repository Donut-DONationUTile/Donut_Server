## Commit convention

다음 컨벤션을 따릅니다.

| 커밋 타입 | 설명                               |
|-----------|----------------------------------|
| `feat`    | 새로운 기능을 추가할 경우                   |
| `fix`     | 버그를 고친 경우                        |
| `style`   | 코드 포맷 변경, 세미 콜론 누락, 코드 수정이 없는 경우 |
| `refactor`| 리팩토링 하는 경우                       |
| `chore`   | 자잘한 수정 사항 반영                     |
| `docs`    | 문서를 수정한 경우                       |
| `test`    | 테스트 추가                           |
